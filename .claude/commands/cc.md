## 📋 工作流程提醒

### 🎯 步骤1：理解任务

1. **检查历史上下文**：查看 `{{PROJECT_ROOT}}/tasks/` 目录，判断是否为继续任务

2. **识别任务类型**：
   - **微任务**：单文件、<30行、无需探索 → 直接执行

   - **复杂任务**：>8文件、架构变更 → 必须创建tasks目录（9章模板）

3. **理解问题现象**：
   - ⚠️ **如果任务描述中提到"日志"或"错误"**，则按以下优先级查阅日志文件：
     - `{{PROJECT_ROOT}}/日志.log` - 主日志文件
   - 使用 Grep/Read 查看相关代码
   - 分析调用链和数据流

4. **标记文档类型**：根据问题性质标记需要查阅的文档


**基岩版实体/NBT 查询**：
```bash
# 在 .claude/docs/bedrock-wiki/ 中搜索

# 示例：查询实体组件

```

**优势**：
- ⚡ **速度快**：本地查询 <1秒（vs WebFetch 5-10秒）
- 🌐 **离线可用**：无需网络连接
- 📖 **完整内容**：可查看完整文档上下文

## ⚠️ CRITICAL 规范提醒

在开发过程中必须遵守以下CRITICAL规范（详见 `markdown/开发规范.md`）：

### ⛔ 规范1: System生命周期

**禁止:**
- ❌ 不调用 `self.Create()` - 会导致事件注册失败

**应该:**
- ✅ 在 `__init__` 中手动调用 `self.Create()`
- 原因: 网易引擎不会自动调用 `Create()`，必须手动触发

### ⛔ 规范2: 模块导入规范

**禁止:**
- ❌ 使用相对路径导入（如 `from ..utils import xxx`）

**应该:**
- ✅ 使用绝对路径导入（如 `from modMain.utils import xxx`）
- 原因: 网易引擎的Python环境不支持相对导入
